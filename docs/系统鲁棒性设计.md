# Luwu OS 系统鲁棒性设计

## 概述

树莓派 CM5 在嵌入式部署中面临两大风险：**电源不稳（欠压/断电）** 和 **系统卡死**。Luwu OS 通过四层防护机制，确保设备在恶劣电源环境下稳定运行、数据不丢失、问题可追溯。

---

## 防护体系

```
第一层: commit=1 + fsck     ← 断电后自动修复文件系统
第二层: 欠压监控 → shutdown  ← 主动检测欠压，干净关机
第三层: 硬件看门狗           ← 系统卡死自动重启
第四层: 持久化日志           ← 出问题可追溯根因
```

### 三层故障场景与对应保护

| 场景 | 现象 | 保护机制 |
|------|------|---------|
| 突然断电 | 瞬间没电 | `commit=1` 最多丢 1 秒数据 + `fsck` 启动时自动修复 |
| 电压不稳 | 系统还能跑但出错风险高 | 欠压监控 10 秒检测 → `shutdown` 干净关机 |
| 系统卡死 | 内核无响应 | 看门狗 15 秒超时 → 硬件 reset |

## 第一层：文件系统保护

### commit=1（ext4 挂载选项）

修改 `/etc/fstab`，在根分区加入 `commit=1`：

```
PARTUUID=xxx  /  ext4  defaults,noatime,commit=1  0  1
```

- ext4 默认 `commit=5`（5 秒刷一次 journal）
- `commit=1` 将刷盘间隔缩短到 1 秒
- 突然断电时最多丢失 1 秒内的写入数据
- 性能影响：极轻微（嵌入式场景基本无感）

### 定期 fsck

```
tune2fs -c 5 /dev/mmcblk0p2
```

- 每 5 次挂载自动执行文件系统检查
- 配合内核启动参数 `fsck.repair=yes`（已配置），检测到错误自动修复
- 首次启动多耗时 2~5 秒

## 第二层：欠压监控与保护关机

### 原理

通过 `vcgencmd get_throttled` 读取 CM5 的电源状态：

```
bit 0 (0x1): 当前是否欠压
bit 16 (0x10000): 是否曾发生过欠压
```

### 脚本逻辑 (`luwu-undervolt-monitor.sh`)

```
每 2 秒检查一次
  ├─ 当前欠压 → 计数器 +1
  │   ├─ 首次检测 → wall 警告所有终端 + 写状态文件
  │   └─ 累计 ≥ 10 秒 → shutdown -h now
  └─ 电压正常 → 计数器归零
```

### 状态文件

- `/tmp/luwu_undervolt_status`
  - `OK` — 电压正常
  - `UNDERVOLT` — 当前欠压中
  - `SHUTDOWN` — 已触发保护关机

上层应用可读取此文件，在 UI 上显示警告通知。

### 为什么是 10 秒？

- 短于 10 秒：电源线接触不良的瞬时波动，不应误关机
- 长于 10 秒：持续的欠压，电源确实有问题，趁系统还能跑赶紧干净关机
- 10 秒窗口内系统仍有写操作，但 `commit=1` 保证最多丢 1 秒

## 第三层：硬件看门狗

### 原理

CM5 的 BCM2712 SoC 内置硬件看门狗定时器。系统正常运行时会定期"喂狗"，一旦系统卡死停止喂狗，定时器到期后硬件强制重启。

### 配置 (`/etc/watchdog.conf`)

```
watchdog-device = /dev/watchdog
watchdog-timeout = 15    # 15 秒不喂狗则重启
interval = 5             # 每 5 秒喂一次
max-load-1 = 24          # 负载过高时不喂（触发重启）
```

### 与欠压监控的互补

- 电压慢慢降低 → 欠压监控先检测到 → 干净 `shutdown`
- 电压骤降导致内核卡死 → 欠压监控也卡住了 → 看门狗 15 秒后硬件 reset

## 第四层：持久化系统日志

### 配置

```
mkdir -p /var/log/journal
sed -i 's/#Storage=auto/Storage=persistent/' /etc/systemd/journald.conf
```

- 默认 systemd journal 存储在 `/run/log/journal`（内存，重启丢失）
- 改为 `Storage=persistent` 后写入 `/var/log/journal`（磁盘，重启保留）
- 磁盘占用：通常几十 MB，可配置上限

### 诊断用途

设备出问题后，通过以下命令查询历史：

```bash
# 查看欠压监控日志
journalctl -t undervolt

# 查看上次启动的关机原因
journalctl -b -1 | grep -i shutdown

# 查看文件系统错误
journalctl -b | grep -i ext4

# 查看看门狗事件
journalctl -u watchdog
```

## 部署

所有防护通过 `install.sh` 一键部署：

```bash
cd /home/pi/luwu-os/configs
sudo bash install.sh
```

对应 `install.sh` 步骤：

| 步骤 | 内容 |
|------|------|
| [7/11] | ext4 文件系统加固（fsck + commit=1） |
| [8/11] | 硬件看门狗 |
| [9/11] | 持久化系统日志 |
| [10/11] | 欠压监控与保护关机 |

| 文件 | 用途 |
|------|------|
| `/usr/local/bin/luwu-undervolt-monitor.sh` | 欠压监控脚本 |
